# 2026-05-12 改进总结：P0-P2 阶段

## 一、项目背景与问题分析

### 1.1 用户反馈

用户在 2026-05-11 测试后反馈：
- 前端很粗糙，需要借鉴领先 AI Agent 前端设计
- Pipeline 有问题，Python 反应速度慢
- 知识图谱呈现的不是真正的图谱
- 无法支持用户自己上传文件

### 1.2 代码审查发现

通过对全部源码的逐文件审读，发现以下关键问题：

**前端问题：**
- TypeScript `any` 泛滥，所有页面的 state 和 API 返回值几乎全部使用 `any`
- `api.ts` 拦截器返回类型丢失，调用处不得不用 `as any` 绕过
- `agentAuditApi` 只有 `run` 方法，缺少 `getStatus`，无法追踪审计进度
- `DocumentsPage` 审计类型硬编码为 `deviation`，用户无法选择
- `KnowledgeGraphPage` 名为"知识图谱"却只有状态卡片、文件列表和查询框，无任何图谱可视化
- 文档上传 `accept` 缺少 `.txt`
- 仪表盘缺少"创建审计任务"入口
- 报告无导出功能

**后端问题：**
- `agent_audit.py` 的 `_run_agent_audit` 无超时保护、无并发控制
- `kg.py` 无图谱数据端点、无文件上传端点
- `lightrag_tool.py` 的 `_model` 闭包变量未预加载，导致 Embedding 冷启动慢
- `agent_helpers.py` 使用 `sys.path` 注入导入 agent

## 五、P2 - Pipeline 性能优化 + 后端健壮性

### 5.1 Embedding 模型预加载

**修改文件：** `backend/app/main.py`

在 `startup()` 函数末尾添加后台线程预加载：
```python
def _preload_embedding():
    from agent.tools.lightrag_tool import preload_embedding_model
    loop = asyncio.new_event_loop()
    loop.run_until_complete(preload_embedding_model())
    loop.close()

threading.Thread(target=_preload_embedding, daemon=True).start()
```

**修改文件：** `agent/tools/lightrag_tool.py`

- `_model` 提升为模块级 `_embedding_model` 单例
- 新增 `preload_embedding_model()` 函数
- 确保全局只加载一次

### 5.2 LightRAG 重复文档检测修复

**修改文件：** `agent/tools/lightrag_tool.py`

- `build_index` 添加 `force_rebuild` 参数
- `force_rebuild=True` 时先清理 `lightrag_output/` 目录
- 使用 `shutil.rmtree()` 删除旧索引
- 重置 `_lightrag_instance = None`

**修改文件：** `backend/app/api/kg.py`

- `POST /kg/build` 端点添加 `force` query parameter

### 5.3 任务超时机制

**修改文件：** `backend/app/api/agent_audit.py`

- 导入 `asyncio` 和 `settings`
- `graph.ainvoke()` 包装 `asyncio.wait_for(timeout=timeout_seconds)`
- 超时时间使用 `settings.DOCUMENT_PROCESS_TIMEOUT`（默认 300 秒）
- 超时时标记任务为 FAILED 并记录错误信息
- 新增 `asyncio.TimeoutError` 异常捕获

### 5.4 移除 sys.path 注入（未完成）

计划中提到但未实施：
