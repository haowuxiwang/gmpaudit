## 五、潜在风险

### 技术债务

1. **`agent-audit` API 冗余** — `POST /api/agent-audit/run` 和 `POST /api/audit/tasks` 都创建 AuditTask，代码路径不同。DocumentsPage 已移除调用，但端点仍存在，可能造成维护困惑。
   - **建议**：标记为 deprecated，后续版本移除或合并到 `audit/tasks`。

2. **`resolved_by` 字段死代码** — `RiskAlert` 模型有 `resolved_by` 字段，但 resolve 端点从未写入，始终为 null。
   - **建议**：要么实现用户追踪（需要用户系统），要么移除字段。

3. **Config 掩码启发式** — `_mask_value` 通过 key 名是否含 "key"/"secret" 判断是否掩码，`max_key_count` 这类 key 会被误掩码。
   - **建议**：改用显式标记（如 `_SENSITIVE_KEYS` 集合）。

4. **`PUT /config/{key}` 用 query 参数** — mutation 端点用 query param 传 value，URL 长度有限制。
   - **建议**：改为 request body。

### 架构风险

5. **Backend-Agent `sys.path` 注入** — 后端通过修改 `sys.path` 导入 agent 模块，脆弱且不可测试。
   - **建议**：将 agent 打包为 Python package，或通过 subprocess/HTTP 调用。

6. **LightRAG 依赖本地文件** — 知识图谱索引存储在 `graphrag_index/` 目录，没有数据库持久化，Electron 打包后路径可能变化。
   - **建议**：使用用户数据目录（`app.getPath('userData')`）。

7. **Embedding 模型体积** — BAAI/bge-large-zh-v1.5 约 1.3GB，Electron 打包时需要考虑分发策略。
   - **建议**：首次启动时下载，或提供模型选择（轻量 vs 精度）。

8. **单文件 SQLite** — 生产环境并发写入可能有锁竞争。
   - **建议**：桌面应用场景下风险较低，但需注意 WAL 模式配置。

### 产品风险

9. **法规库覆盖不足** — fallback DB 仅 10 条，LightRAG 索引仅 5 个文档。对真实审计场景远远不够。
   - **建议**：P1 优先扩充，至少覆盖中国 GMP、FDA 21 CFR Part 211、EU GMP Annex。

10. **LLM 输出不可控** — 当前无验证 Agent，LLM 可能生成虚假 findings 或遗漏关键问题。
    - **建议**：实现验证 Agent + 人工审查节点（P1/P3）。

11. **无审计追溯** — 系统不记录"谁在什么时候对哪个告警做了什么操作"，无法满足 GMP 审计追溯要求。
    - **建议**：实现操作日志表（P3）。
