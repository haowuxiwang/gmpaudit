## 四、.doc 文件处理修复（重要 Bug Fix）

### 问题
用户上传 `.doc` 文件时处理失败，连续两轮修复：
1. 第一轮：用 `mammoth` 替代 `python-docx` → 仍失败，mammoth 不支持 OLE2 二进制格式
2. 第二轮：区分 `.doc`（OLE2）和 `.docx`（Open XML），分别用不同工具处理

### 根因
- `.doc` = OLE2 二进制格式（magic bytes: `d0cf11e0a1b11ae1`）
- `.docx` = Open XML（ZIP 压缩格式）
- `mammoth` 和 `python-docx` 都只支持 `.docx`，不支持 `.doc`

### 最终方案
- `file_utils.py`: `.doc` → `word_legacy`，`.docx` → `word`
- `document_processor.py`:
  - `word` 类型 → `mammoth.extract_raw_text()`（处理 .docx）
  - `word_legacy` 类型 → `antiword` 命令行工具（处理 .doc）
- `antiword` 已安装于 `D:\git\Git\mingw64\bin\antiword.exe`，能完美提取中文文本

### 修改的文件
- `backend/app/utils/file_utils.py` — `.doc` 映射改为 `word_legacy`
- `backend/app/services/document_processor.py` — 添加 `subprocess` 调用 antiword
- `backend/requirements.txt` — `python-docx` 替换为 `mammoth>=1.6.0`
- `backend/tests/test_document_processor.py` — 全部 mock 从 DocxDocument 迁移到 mammoth + antiword
