# GMP合规性审计系统 - 项目总结 (2026-05-06)

## 项目架构

```
gmpaudit/
├── backend/                  # Python FastAPI后端
│   ├── app/
│   │   ├── api/              # API路由 (documents, audit, reports, config)
│   │   ├── core/             # 核心配置 (config.py, database.py)
│   │   ├── models/           # 数据模型 (6个模型)
│   │   ├── services/         # 业务服务 (LLM引擎, 审计引擎, 文档处理)
│   │   └── utils/            # 工具函数
│   └── tests/                # 测试用例 (14个)
├── frontend/                 # React + TypeScript + Electron前端
│   ├── src/
│   │   ├── components/       # 公共组件 (Header, Sidebar, Loading)
│   │   ├── pages/            # 页面 (Dashboard, Documents, AuditTasks, Reports, Settings)
│   │   ├── services/         # API服务 (axios)
│   │   └── stores/           # 状态管理 (zustand, 未使用)
│   └── electron/             # Electron主进程
├── config/                   # 配置文件 (.env, audit_rules.json, system_config.json)
├── data/                     # 运行时数据 (数据库, 上传文件, 报告)
├── scripts/                  # 启动脚本 (start.bat, start.sh)
└── docs/                     # 文档
```

## 技术栈

| 层级 | 技术 |
|------|------|
| 后端框架 | FastAPI + SQLAlchemy (异步) |
| 数据库 | SQLite + aiosqlite |
| LLM集成 | DeepSeek / 通义千问 / 智谱GLM (适配器模式) |
| 文档处理 | PyMuPDF (PDF) + python-docx (Word) + PaddleOCR (图片) |
| 前端框架 | React 18 + TypeScript + Ant Design 5 |
| 桌面应用 | Electron 28 |
| 测试框架 | pytest + pytest-asyncio |

## 数据模型

| 模型 | 说明 |
|------|------|
| Document | 文档管理 (上传、处理状态、内容提取) |
| AuditTask | 审计任务 (偏差分析、SOP合规、一致性检查、风险评估) |
| Finding | 审计发现 (类型、严重程度、证据、建议) |
| Report | 审计报告 (全文、摘要、元数据) |
| Configuration | 系统配置 (键值对) |
| RiskAlert | 风险告警 (级别、状态) |

## API端点

- **文档API** (`/api/documents`): 上传、批量上传、列表、详情、处理、删除
- **审计API** (`/api/audit`): 创建任务、任务列表、任务详情、运行任务、获取发现、风险评估
- **报告API** (`/api/reports`): 报告列表、生成报告、报告详情
- **配置API** (`/api/config`): 配置管理、可用模型列表

## 已修复的问题

### 后端修复

1. **数据库路径问题** (`backend/app/core/config.py`)
   - 问题: 相对路径`./data/...`相对于backend目录解析，但数据目录在项目根目录
   - 修复: 计算`PROJECT_ROOT`并使用绝对路径

2. **Report模型字段名** (`backend/app/api/reports.py`)
   - 问题: 使用`metadata=`而非`report_metadata=`
   - 修复: 改为`report_metadata=`

### 前端修复

3. **npm依赖冲突** (`frontend/package.json`)
   - 问题: `react-scripts@5.0.1`要求`typescript@^3.2.1 || ^4`，与`^5.3.0`冲突
   - 修复: 降级typescript到`^4.9.5`，创建`.npmrc`设置`legacy-peer-deps=true`

4. **缺少HTML模板** (`frontend/public/index.html`)
   - 问题: react-scripts需要public/index.html作为入口
   - 修复: 创建标准HTML模板

5. **ajv模块缺失**
   - 问题: webpack依赖的ajv模块未正确安装
   - 修复: 显式安装`ajv`包

6. **TypeScript类型错误** (多个页面文件)
   - 问题: API拦截器返回`response.data`但TS推断为`AxiosResponse`
   - 修复: 添加`const result: any =`类型断言

## 启动方式

### 后端
```bash
cd backend
pip install -r requirements.txt
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 前端
```bash
cd frontend
npm install
npm start
```

### 一键启动 (Windows)
```bash
scripts/start.bat
```
